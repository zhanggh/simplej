支付系统服务说明

第三方支付平台服务 功能
mch-gateway 支付网关服务，提供TO B业务的支付相关服务，如:统一下单、绑卡、发短信、解绑、快捷支付、支付结果查询、 退款、退款结果查询、代扣、代扣结果查询、代付、代付结果查询、实名验证、下载对账单等服务， 更多使用案例见：mch-gateway使用手册
payment-business 支付应用子系统，由于仅实现了一些较为简单标准的支付功能，故中间层只设计了一个统一的支付应用服务， 在业务较为丰富复杂的场景下，支付应用中间层实际上是可以划分成多个不同领域的子服务系统，更多使用案例见：payment-business使用手册
payment-admin 支付运营管理平台，提供常规的商户管理、产品管理、计费管理、交易管理、结算管理、协议管理、账户管理等基础功能， 更多使用案例见：payment-admin 使用手册
payment-userinfo 支付平台商户&用户服务，提供基础的商户&用户入户查询修改等基础功能， 更多使用案例见：payment-userinfo 使用手册
payment-accounting 支付平台账户系统（包含账户&科目），更多使用案例见：payment-accounting使用手册
payment-settlement 支付平台结算管理服务，更多使用案例见：payment-settlement使用手册
payment-engine 支付平台核心引擎，上游来着payment-business，下游去payment-channel， 更多使用案例见：payment-engine 使用手册
payment-risk 支付平台风控系统，提供事前事中事后3个阶段的风险控制能力， 更多使用案例见：payment-risk 使用手册
payment-channel 支付平台统一渠道网关系统，负责对接外部渠道，包括银行，短信运营商，营销平台等， 更多使用案例见：payment-risk 使用手册
payment-agreement 支付平台商户&用户协议管理子系统，所有商户&用户相关的签约/绑卡协议信息都由该系统统一登记管理， 更多使用案例见：payment-agreement 使用手册
payment-router 支付平台智能路由子系统，为支付提供渠道路由的能力， 更多使用案例见：payment-router 使用手册
